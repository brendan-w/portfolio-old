This python module serves as a back-end for projects using sensor data from a car's OBD-II port. It seemlessly handles the decoding of data from the car, and returns values with units to the programmer. Derived from the popular pyobd project, this package attempts to do a better job of decoupling the serial operations from the UI code.

Since the initial fork of pyobd, the code has been substantially overhauled. The library now makes better use of the Python stdlib, supports more commands, and presents a simple API for accessing the car's data. Commands (sensors) can now be accessed by name, or Parameter ID. Threaded callbacks can also be subscribed, allowing for asynchronous applications.

Here are a few of the currently supported commands (for a full list, see the GitHub Wiki):

  • Calculated Engine Load
  • Engine Coolant Temperature
  • Fuel Pressure
  • Intake Manifold Pressure
  • Engine RPM
  • Vehicle Speed
  • Timing Advance
  • Intake Air Temp
  • Air Flow Rate (MAF)
  • Throttle Position
  • Engine Run Time
  • Fuel Level Input
  • Number of warm-ups since codes cleared
  • Barometric Pressure
  • Ambient air temperature
  • Commanded throttle actuator
  • Time run with MIL on
  • Time since trouble codes cleared
  • Hybrid battery pack remaining life
  • Engine fuel rate
